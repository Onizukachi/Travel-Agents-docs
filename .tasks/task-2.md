# PDF-чек: НДС 5% для сервисного сбора

## Бизнес контекст для задачи

Проект — travel-агрегатор. При переходе на новое юрлицо в заказах по умолчанию появляется допуслуга «Сервисный сбор». 

Нужно корректно отображать такие чеки в PDF:
- под товарной позицией сервисного сбора выводить строку `НДС 5%`;
- в блоке summary после `ИТОГО` выводить `СУММА НДС 5%` и сумму справа;
- если в чеке нет строки с НДС 5%, оставлять старое поведение (`НДС НЕ ОБЛАГАЕТСЯ`).

Из входных уточнений:
- определение сервисного сбора для PDF делаем по `vat` в данных чека, а не по запросам к `OrderExtra`;
- для строки с `vat=5` убираем текст `в т.ч. НДС НЕ ОБЛАГАЕТСЯ` в строке цены;
- НДС считаем по формуле `sum(vat=5) * 5 / 105`, округление до 2 знаков;
- в текущем бизнес-потоке ожидается одна строка с НДС 5%, но реализация должна корректно агрегировать сумму по всем строкам `vat=5`.

## Ключевые решения и обоснования

1. Распознавание НДС 5% в PDF делать по `vat` из фискального ответа.
- Причина: это финальные фискальные данные, не зависит от состояния локальных `OrderExtra`, не требует дополнительных SQL-запросов при генерации PDF.

2. Расширить парсинг чека так, чтобы `vat` попадал в каждую строку `receipt_data[:line_items]`.
- Причина: текущий `FiscalParser#format_line_item` не сохраняет `vat` на уровне line item, а PDF работает именно с `receipt_data[:line_items]`.

3. Логику отображения НДС держать в общем слое, который используют и `FiscalPdfV2`, и `FiscalPartnerPdf`.
- Причина: требование распространяется на оба формата, и нужно избежать расхождений поведения.

4. Summary строить условно:
- если есть `vat=5` в строках, показывать `СУММА НДС 5%` + рассчитанную сумму;
- если нет — показывать `НДС НЕ ОБЛАГАЕТСЯ`.
- Причина: строгое соответствие заданному UX и текущим правилам бизнеса.

## Заметки для восстановления сессии

Текущее состояние и уже сделанная аналитика:
- Основной PDF-роутинг: `FiscalPdf.select_drawer` выбирает `FiscalPdfV2` или `FiscalPartnerPdf`.
- Формирование товарных строк: `app/pdfs/receipt_line_items.rb`.
- Формирование summary: `app/pdfs/fiscal_pdf_v2.rb#summary_table_data`.
- В `FiscalProcessor::Uniteller` уже есть бизнес-логика НДС 5% для service fee на этапе фискализации, но PDF сейчас ее не использует напрямую.
- В `FiscalParser#format_line_item` сейчас отсутствует `vat` внутри каждого line item, из-за этого PDF не может надежно различать строки `vat=5`.

Критически важные решения:
- Не определяем service fee через `OrderExtra.exists?` в PDF.
- Источник истины для PDF — `vat` в данных фискального чека.
- Формат summary по условию:
  - есть `vat=5` -> `СУММА НДС 5%` + сумма справа;
  - нет `vat=5` -> `НДС НЕ ОБЛАГАЕТСЯ`.

Точка продолжения при старте реализации:
1. Начать с `FiscalParser` и теста ключей line item (`:vat` в parsed line item).
2. Затем обновить `ReceiptLineItems` (строка `НДС 5%` + формат цены).
3. После этого обновить `FiscalPdfV2#summary_table_data` и проверить применимость к `FiscalPartnerPdf`.
4. Завершить корректировкой высоты PDF и тестами.

Стоп-фактор перед реализацией:
- Ждем отдельный сигнал пользователя на старт реализации.
